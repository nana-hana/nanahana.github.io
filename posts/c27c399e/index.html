MyBatis 概述

MyBatis
Java
发布日期:   2019-09-22
更新日期:   2020-10-29
文章字数:   3.5k
阅读次数:  

MyBatis 概述

  1. MyBatis 封装了 JDBC，让开发无需花费精力去处理加载驱动、创建连接、创建 Statement 等繁杂的过程。
  2. MyBatis 通过 XML 或注解的方式将要执行的各种 Statement 配置起来，并通过 Java 对象 和 Statement 中 SQL 的动态参数进行映射生成最终执行的 SQL 语句，最后由 MyBatis 框架执行 SQL 并将结果映射为 Java 对象并返回。
  3. MyBatis 采用 ORM 思想解决了实体和数据库映射的问题，因此不需要直接操作 JDBC API 便可以完成对数据库的持久化操作。

mapper.xml 常用标签

mapper.xml 映射文件是 MyBatis 的核心文件，它存储了操作数据库的 sql 语句。

resultMap

resultMap 可以实现将查询结果映射为复杂类型的 POJO。

参数描述
id用于定义 resultMap 的 id。
type复杂类型的实体全限定名。
result对数据库字段和 Java 变量进行映射。
column: 数据库字段。
jdbcType: 数据库字段类型。
property：java 变量。
collection关联另一个 resultMap，常用于一对多查询。
association关联另一个 resultMap，常用于一对一查询。
<resultMap id="UserResultMap" type="com.kuukokawaii.user.entity.User">
    <id column="id" jdbcType="INTEGER" property="id"/>
    <result column="name" jdbcType="VARCHAR" property="name"/>
    <result column="birthday" jdbcType="DATE" property="birthday"/>
    <result column="sex" jdbcType="VARCHAR" property="sex"/>
    <result column="address" jdbcType="VARCHAR" property="address"/>
    <association property="account" resultMap="AccountResultMap"/>
</resultMap>
<resultMap id="AccountResultMap" type="com.kuukokawaii.user.entity.Account">
    <id column="id" jdbcType="INTEGER" property="id"/>
    <result column="username" jdbcType="VARCHAR" property="username"/>
    <result column="password" jdbcType="VARCHAR" property="password"/>
</resultMap>

sql

sql 被用来定义可重用的 sql 代码段，可以通过<include refid='sql 的 id' />标签进行调用。

<sql id="User_Column_List">
    <!--@sql select -->
    user.id, name, birthday, sex, address
    <!--@sql from user -->
</sql>

if

if 提供了一种可选的查找文本功能，符合条件即执行。

  1. #{} 是指引用传入的 Java 变量，jdbcType 可以省略。
  2. test=”” 也是指传入的 Java 变量。
<select id="getUser" parameterType="string" resultMap="UserResultMap">
    select
    <include refid="User_Column_List"/>
    from user
    where name = #{name, jdbcType=VARCHAR}
    <if test="sex != null">
        and sex = #{sex}
    </if>
</select>

choose

choose 从多个条件中挑选一个，类似于 switch 语句。when 是条件判断，otherwise 不匹配 when 的其他选项。

<select id="getUser" parameterType="string" resultMap="UserResultMap">
    select
    <include refid="User_Column_List"/>
    from user
    where name = #{name}
    <choose>
        <when test="sex != null">
            and sex = #{sex}
        </when>
        <when test="birthday != null">
            and birthday = #{birthday}
        </when>
        <otherwise>
            and address = #{address}
        </otherwise>
    </choose>
</select>

trim, where, set

当使用 if 进行条件匹配时，如果一个条件都不匹配则语句末尾可能以 where 结尾导致 SQL 报错。若匹配的条件非第一项而是其他项则会导致 where and xxx 的语句情况同样会报错。因此 MyBatis 的 where 标签会自动处理该情况，set 标签与 where 标签作用相同。当 where 和 set 标签没有发挥作用时可以通过 trim 自定义格式执行。

<select id="getUser" parameterType="string" resultMap="UserResultMap">
    select
    <include refid="User_Column_List"/>
    from user
    <where>
        <if test="name != null">
            name = #{name}
        </if>
        <if test="sex != null">
            and sex = #{sex}
        </if>
    </where>
</select>
<!-- 
    where 等价于 trim 的如下情形
    prefixOverrides: 会移除所有 prefixOverrides 属性中指定的内容(注意此例中的空格是必要的)
-->
<trim prefix="WHERE" prefixOverrides="AND |OR ">
    <!-- 内容 -->
</trim>
<update id="updateUser" parameterType="com.kuukokawaii.user.entity.User">
    update user
    <set>
        <if test="id != null">
            user.id = #{id},
        </if>
        <if test="name != null">
            name = #{name},
        </if>
        <if test="birthday != null">
            birthday = #{birthday},
        </if>
        <if test="sex != null">
            sex = #{sex},
        </if>
        <if test="address != null">
            address = #{address},
        </if>
    </set>
    where user.id = #{id}
</update>
<!-- set 等价于 trim 的如下情形 -->
<trim prefix="SET" prefixOverrides=",">
    <!-- 内容 -->
</trim>

foreach

foreach 用于对一个集合进行遍历操作。

参数描述
item表示集合中每一个元素进行迭代时的别名。
index指定一个别名，用于获取迭代过程中的位置(从下标 0 开始)。
open迭代开始以什么符号开始。
separator迭代时以什么符号作为分隔符。
close迭代结束以什么符号结束。
<select id="selectUserIn" resultMap="UserResultMap">
    select * from user
    where user.id in
    <foreach item="item" index="index" collection="list" open="(" separator="," close=")">
        #{item}
    </foreach>
</select>

bind

bind 元素可以从 OGNL 表达式中创建一个变量并将其绑定到上下文。

<select id="selectUsersLike" resultMap="UserResultMap">
    <!-- 假设传入一个 user 对象，其中有 name 字段 -->
    <bind name="pattern" value="'%' + name + '%'" />
    select * from user
    where name like #{pattern}
</select>

select, delete, update, insert

insert、select、update 和 delete 分别是 MyBatis 的 CRUD 语句。

参数描述
idid 需要与 DAO 中的方法名对应。
parameterType表传入的参数类型。
resultType表示返回类型(与 resultMap 二选一)，如果返回的是集合则填写集合所包含的类型。
resultMap表示返回类型(与 resultType 二选一)，返回定义的 resultMap。
useGeneratedKeys更新后返回主键(默认 false)
keyProperty指定在 Java 实体中对应的主键变量名。
keyColumn当主键列不是表中的第一列的时候需要设置，用于指定数据库的主键（默认数据库第一列为主键）。
<select id="listUser" resultMap="UserResultMap">
    select
    <include refid="User_Column_List"/>,
    from user
</select>
<delete id="deleteUser" parameterType="int">
    delete
    from user, account
    using user, account
    where user.id = account.id
    and user.id = #{id};
</delete>
<update id="updateUser" parameterType="com.kuukokawaii.user.entity.User">
    update user
    <set>
        <if test="id != null">
            user.id = #{id},
        </if>
        <if test="name != null">
            name = #{name},
        </if>
    </set>
    where user.id = #{id}
</update>
<insert id="createUser" parameterType="com.kuukokawaii.user.entity.User" useGeneratedKeys="true" keyProperty="id">
    insert into user
    <trim prefix="(" suffix=")" suffixOverrides=",">
        <if test="name != null">
            name,
        </if>
    </trim>
    <trim prefix="values (" suffix=")" suffixOverrides=",">
        <if test="name != null">
            #{name},
        </if>
    </trim>
</insert>

多表操作

  1. 一个用户拥有多个账号(一对多使用collection标签)。
  2. 一个账号只能属于一个用户(一对一使用association标签)。
  3. 一个用户可以有多个角色，一个角色可以有多个用户(多对多在各自的 mapper.xml 文件中使用 collection 关联对方的 resultMap 即可)。
/**
 * @Author 喵粮都输光了
 * @Date 2019/9/23 16:28
 * @Description 用户实体
 */
@Data
public class User implements Serializable {
    private Integer id;
    private String name;
    private List<Account> accounts;
}
/**
 * @Author 喵粮都输光了
 * @Date 2019/9/30 14:53
 * @Description 账户实体
 */
@Data
public class Account implements Serializable {
    private Integer id;
    private String username;
    private String password;
    private User user;
}

UserMapper.xml 一对多设置。

<resultMap id="UserResultMap" type="com.kuukokawaii.user.entity.User">
    <id column="id" jdbcType="INTEGER" property="id"/>
    <result column="name" jdbcType="VARCHAR" property="name"/>
    <collection property="accounts" resultMap="AccountResultMap"/>
</resultMap>
<resultMap id="AccountResultMap" type="com.kuukokawaii.user.entity.Account">
    <id column="id" jdbcType="INTEGER" property="id"/>
    <result column="username" jdbcType="VARCHAR" property="username"/>
    <result column="password" jdbcType="VARCHAR" property="password"/>
</resultMap>

AccountMapper.xml 一对一设置。

<resultMap id="AccountResultMap" type="com.kuukokawaii.user.entity.Account">
    <id column="id" jdbcType="INTEGER" property="id"/>
    <result column="username" jdbcType="VARCHAR" property="username"/>
    <result column="password" jdbcType="VARCHAR" property="password"/>
    <association property="user" resultMap="AccountResultMap"/>
</resultMap>
<resultMap id="UserResultMap" type="com.kuukokawaii.user.entity.User">
    <id column="id" jdbcType="INTEGER" property="id"/>
    <result column="name" jdbcType="VARCHAR" property="name"/>
</resultMap>

延迟加载

查询用户时，如果不需要查询该用户相关联的账号则可以使用延迟加载(懒加载、按需加载)。如果需要查询关联的账号则使用立即加载。

一对多，多对多: 通常情况下都是采用延迟加载。
一对一: 通常情况下都是采用立即加载。

  1. 在 MyBatis 配置文件中开启延迟加载开关。

     <settings>
     <!-- 开启延迟加载开关 -->
     <setting name="lazyLoadingEnabled" value="true"/>
     <!-- 用于控制具有懒加载特性的对象的属性的加载情况 -->
     <setting name="aggressiveLazyLoading" value="false"/>
 </settings>

  2. 一对多延迟加载: 在需要延迟加载的 collection 标签中设置需要延迟加载的 select 方法。例如此处为查询用户延迟加载用户关联的账号。

     <resultMap id="UserResultMap" type="com.kuukokawaii.user.entity.User">
     <id column="id" jdbcType="INTEGER" property="id"/>
     <result column="name" jdbcType="VARCHAR" property="name"/>
     <!--
         延迟加载一对多。
         select：表示调用其他查询（可以是其他mapper中的查询），此处为查询用户的唯一标识。
         column：用户根据 id 查询时，所需要的参数的值。即为 select 中查询提供所需的 id 进行查询。
     -->
     <collection column="id" property="accounts" select="com.kuukokawaii.user.dao.AccountDao.getAccount"/>
 </resultMap>

  3. 一对一延迟加载：与一对多延迟加载类似。

     <resultMap id="AccountResultMap" type="com.kuukokawaii.user.entity.Account">
     <id column="id" jdbcType="INTEGER" property="id"/>
     <result column="username" jdbcType="VARCHAR" property="username"/>
     <result column="password" jdbcType="VARCHAR" property="password"/>
     <!--
         延迟加载一对一（多对一）。
         select：表示调用其他查询（可以是其他mapper中的查询），此处为查询用户的唯一标识。
         column：用户根据 id 查询时，所需要的参数的值。即为 select 中查询提供所需的 id 进行查询。
     -->
     <association column="id" property="user" select="com.kuukokawaii.user.dao.UserDao.getUser"/>
 </resultMap>

缓存

缓存适用于经常查询且不经常改变的数据，且数据的正确与否对最终结果影响不大。MyBatis 默认定义了一级缓存和二级缓存，极大的提升了查询效率。

一级缓存

MyBatis 的一级缓存指的是 SqlSession 对象的缓存。当执行查询之后，查询的结果会同时存入到 SqlSession 中的一块 Map 结构的存储区域中。当我们再次执行查询同样的数据时，MyBatis 会先去 SqlSession 中查询是否拥有该数据，有则直接从缓存中获取。需要注意的是当调用 SqlSession 对象的修改、添加、删除、commit()、close()等方法时，MyBatis 的一级缓存将会被清空。

二级缓存

MyBatis 的二级缓存指的是 SqlSessionFactory 对象的缓存。由同一个 SqlSessionFactory 对象创建的 SqlSession 对象将共享该缓存。不同于一级缓存，二级缓存需要手动配置，分为三个步骤:

  1. MyBatis 框架开启二级缓存支持。

     <settings>
     <!-- 开启二级缓存支持。（可以不进行配置，默认 true。） -->
     <setting name="cacheEnabled" value="true"/>
 </settings>

  2. 开启当前映射文件的二级缓存支持。<cache-ref/>可以引用其它映射文件的缓存。

     <!-- 在 mapper 标签下开启二级缓存支持， -->
 <cache/>

  3. 开启当前方法的二级缓存支持。

     <!-- 通过 userCache 属性开启二级缓存 -->
 <select id="findUsers" resultMap="UserResultMap" useCache="true">
     select
     <include refid="User_Column_List"/>
     from user
 </select>

注解开发

在 MyBatis 中如果使用注解进行开发(代替了 mapper.xml)，那么就不能使用 XML 的方式进行开发。

select 注解开发

/**
 * 查询指定的用户
 * @param id 需要查询的 user id
 * @return 返回查找到的 user
 */
@Select("select * from user where id = #{id};")
User getUser(Integer id);

insert 注解开发

/**
 * 插入一个用户
 * @param user 需要插入的 user
 * @return 返回插入结果
 */
@Insert("insert into user (id, name) values (#{id}, #{name});")
int createUser(User user);

update 注解开发

/**
 * 更新已给用户
 * @param user 需要更新的 user
 * @return 返回更新结果
 */
@Update("update user set id = #{id}, name = #{name} where id = #{id};")
int updateUser(User user);

delete 注解开发

/**
 * 删除指定用户
 * @param id 需要删除用户的 id
 * @return 返回删除结果
 */
@Delete("delete from user where id = #{id};")
int deleteUser(Integer id);

resultMap 注解开发

MyBatis 提供了@Results、@Result和@ResultMap三个注解用于调用和实现 XML 配置中的 ResultMap 标签。

  1. @Results: 相当于 XML 配置中的ResultMap标签；
  2. @Result: 相当于 XML 配置中ResultMap的子标签result；
  3. @ResultMap: 相当于 XML 配置中方法的resultMap属性，通过填入ResultMap的id与之对应。
/**
 * 查找所有用户
 * @return 返回 User 集合
 */
@Select("select * from user")
@Results(id = "UserResultMap", value = {
    @Result(id = true, column = "id", jdbcType = JdbcType.INTEGER, property = "id"),
    @Result(column = "name", jdbcType = JdbcType.VARCHAR, property = "name")
})
List<User> listUser();
/**
 * 查询指定的用户
 * @param id 需要查询的 user id
 * @return 返回查找到的 user
 */
@Select("select * from user where id = #{id};")
@ResultMap("UserResultMap")
User getUser(Integer id);

多表查询及延迟加载注解开发

MyBatis 提供了@One和@Many两个注解用于实现 XML 配置中的association标签和collection标签。其中fetchType属性可以选择快速加载或延迟加载。

一对一查询:

/**
 * 查找所有 Account(立即加载)
 * @return 返回Account集合
 */
@Select("select * from account")
@Results(id = "AccountResultMap", value = {
    @Result(id = true, column = "id", jdbcType = JdbcType.INTEGER, property = "id"),
    @Result(column = "username", jdbcType = JdbcType.VARCHAR, property = "username"),
    @Result(column = "password", jdbcType = JdbcType.DATE, property = "password"),
    @Result(column = "id", jdbcType = JdbcType.INTEGER, property = "user", one = @One(
        select = "com.kuukokawaii.user.dao.UserDao.getUser", fetchType = FetchType.EAGER
    ))
})
List<Account> listAccount();

一对多查询:

/**
 * 查找所有 User(延迟加载)
 * @return 返回User集合
 */
@Select("select * from user")
@Results(id = "UserResultMap", value = {
    @Result(id = true, column = "id", jdbcType = JdbcType.INTEGER, property = "id"),
    @Result(column = "name", jdbcType = JdbcType.VARCHAR, property = "name"),
    @Result(column = "birthday", jdbcType = JdbcType.DATE, property = "birthday"),
    @Result(column = "sex", jdbcType = JdbcType.VARCHAR, property = "sex"),
    @Result(column = "address", jdbcType = JdbcType.VARCHAR, property = "address"),
    @Result(column = "id", property = "accounts", many = @Many(
        select = "com.kuukokawaii.user.dao.AccountDao.getAccount", fetchType = FetchType.LAZY
    ))
})
List<User> listUser();

二级缓存注解开发

MyBatis 提供了@CacheNamespace用于实现 XML 配置中cache标签的功能，需要注意的是该注解使用于Dao接口上而非方法上。二级缓存可以在 MyBatis 配置文件中进行开启(默认开启)。

@CacheNamespaceRef相当于实现 XML 配置中<cache-ref/>标签的功能。

参数描述
readWrite读写模式，若为 True 则必须实现序列化接口。
flushInterval刷新时间(单位是毫秒)，这里的刷新是指缓存数据的有效期。
implementation缓存实现自定义对象。
eviction缓存策略的算法。
LRU(默认): 最近最少使用，移除最长时间不被使用的对象。
FIFO: 先进先出，按对象进入缓存的顺序来移除它们。
SOFT: 软引用，移除基于垃圾回收器状态和软引用规则的对象。
WEAK: 弱引用，更积极地移除基于垃圾收集器和弱引用规则的对象。
size设置缓存存放多少个元素。
blocking防止缓存击穿，对应的实现类为 BlockingCache.putObject。
@CacheNamespace(blocking = true)
public interface UserDao {
    // 内容省略
}
